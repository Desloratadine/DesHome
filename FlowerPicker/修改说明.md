# 《层渊花园》策划案修改说明

## 📋 **修改概述**

根据新的游戏策划案《层渊花园》，对原有"草皮花园"代码进行了全面重构，实现了以资源采集与路径探索为核心的回合制策略游戏。

## 🎮 **核心机制修改**

### 1. **多层系统（4层深渊）**
- **层数设计**：浅草（1）、湿地（2）、荆棘（3）、深渊（4）
- **难度递增**：
  - 点亮地块所需花朵：3→4→5→6
  - 新地块花朵生成：3~6→2~5→2~4→1~3
- **层级过渡**：支付怪物代价后进入下一层，背包保留

### 2. **阶梯地块系统**
- 每层有一个隐藏的**阶梯地块**（初始灰色）
- 点亮阶梯地块触发**守关怪物**
- 使用指南针（消耗2朵花）定位阶梯方向
- 指南针精度随使用次数提升：大致方向→精确距离→精确坐标

### 3. **怪物选择机制**
- **守关怪物**：点亮阶梯地块后触发
- **索要代价**：背包花朵数量的50%（向上取整）
- **玩家选择**：
  - **支付代价**：扣除花朵，进入下一层，获得宝藏碎片（前三层）
  - **拒绝支付**：怪物摧毁一个随机已点亮地块，进入**游击模式**

### 4. **游击模式**
- 拒绝守关怪物后触发
- 每点亮**2块新地块**再次遇到游击怪物
- 游击怪物同样索要背包花朵的50%
- 可随时返回阶梯地块支付代价进入下一层

### 5. **地块状态系统**
- **灰色未点亮**：不可采集，不可通行
- **绿色已点亮**：可采集花朵，可立足
- **枯萎地块**：被怪物摧毁的地块，需消耗花朵恢复
- **阶梯地块**：特殊标记（🪜），通往下一层的入口

### 6. **胜利条件**
- **主要目标**：穿越4层深渊，在第四层找到"大地之心"
- **碎片收集**：前三层各获得一个宝藏碎片
- **替代胜利**：游击模式下点亮本层所有非枯萎地块也可进入下一层（无碎片）

## 🔧 **技术实现修改**

### 1. **游戏常量重构**
```javascript
const TOTAL_LAYERS = 4; // 总层数
const FLOWERS_TO_LIGHT_BASE = 3; // 第一层点亮消耗
const MONSTER_INTERVAL = 2; // 游击模式触发间隔
// 层数相关数值通过函数动态计算
```

### 2. **状态管理系统**
- 新增`currentLayer`、`treasureFragments`、`inGuerrillaMode`等状态
- 阶梯地块标记`isStair`、枯萎状态`withered`
- 游击模式计数器`guerrillaTriggerCount`

### 3. **UI界面更新**
- **标题**："草皮花园" → "层渊花园"
- **进度显示**：层数指示器（1/4）和碎片计数（0/3）
- **怪物弹窗**：支付/拒绝双按钮选择
- **地块视觉**：阶梯标记、枯萎状态、恢复提示

### 4. **核心函数新增/修改**
- `getFlowersToLight()`：根据层数计算点亮消耗
- `getNewTileFlowersMin/Max()`：根据层数计算花朵生成范围
- `triggerStairMonster()`：守关怪物触发
- `triggerGuerrillaMonster()`：游击怪物触发
- `payMonster()` / `refuseMonster()`：怪物选择处理
- `advanceToNextLayer()`：进入下一层逻辑
- `restoreWitheredTile()`：恢复枯萎地块

### 5. **绘制系统增强**
- 阶梯地块特殊标记（🪜）
- 枯萎地块视觉表现（💀标记）
- 层数主题颜色区分（未来可扩展）

## 📊 **数值平衡参考**

### 第一层（浅草）
- 点亮消耗：3朵花
- 新地块花朵：3~6朵
- 背包容量：6格
- 指南针消耗：2朵花
- 怪物索要：背包花朵的50%

### 难度曲线
- 每层点亮消耗+1朵
- 每层花朵生成-1朵（最小范围）
- 背包容量可通过道具扩展（最多8格）
